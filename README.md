# Surge Hiring Toolkit — Hybrid Hub Model

## Overview
Brief description of the problem (surge hiring, clearance-heavy environment, remote readiness) and your solution (2-hub spreadsheet system + 4 supporting documents + automation rules).

## Architecture
- 2 spreadsheet hubs (Recruiting & Security, Operations & Employee).
- 4 core documents (Surge Playbook, Operations Guide, New Hire Kit, After-Action Report).
- Extra supporting documents (change log, risk approach, etc.).
- No macros, no VBA — just formulas, dropdowns, and conditional formatting.

## Features
- Automated dashboards (pipeline, headcount, QA, risk).
- Alerts for stuck candidates, stalled clearances, overdue tasks, low QA, retention risk, remote readiness.
- Clean handoff from candidate to employee.
- Version-control-friendly structure (2 main spreadsheets).

## Repository Structure
Explain the folders: `docs/`, `data/`, `templates/`, `notion/`.

## How to Use
1. Download the templates from `templates/`.
2. Populate sample data or your own data in the hub spreadsheets.
3. Use the data dictionaries in `data/` to understand fields and constraints.
4. Review `docs/01_architecture_overview.md` for the full system design.

## Portfolio Notes
- Designed as a no-code automation toolkit for Excel/Google Sheets/SharePoint.
- Focused on operationalizing recruiting, security, and employee lifecycle in surge environments.
